# 1HandIndia Workspace Skill Loading Guide

This guide explains which skills should be used when working on the 1HandIndia marketplace project.

## 1. Always Load Project Context First

Before planning or coding, read:

- `AGENTS.md`
- `README.md`
- `docs/IndiHub_Final_Scope_Requirement_Confirmation_Phase1.md`
- `docs/IndiHub_PROJECT_SCOPE_AND_REQUIREMENTS.md`
- `docs/IndiHub_BUILD_BLUEPRINT_MNC_PORTAL.md`
- `docs/IndiHub_FINAL_TECH_STACK_LOCK.md`
- `docs/IndiHub_TECH_STACK_DECISION.md`
- `docs/IndiHub_BRAND_DIRECTION.md`
- `docs/IndiHub_UI_SCREEN_LIST_AND_DATABASE_PLAN.md`
- `.agents/skills/beeshub-marketplace/SKILL.md`

## 2. Skills to Use by Task

### Client Documents and Scope Writing

Use `technical-writer`.

Use it for:

- Client proposals.
- Scope confirmation.
- Requirement documents.
- WhatsApp sharing text.
- User manuals.
- Handoff documents.

### Planning and Milestones

Use `project-planner`.

Use it for:

- Roadmap.
- Milestones.
- Work breakdown.
- Timeline.
- Dependencies.
- Risk planning.

### Full-Stack Build Work

Use `fullstack-developer`.

Use it for:

- Architecture.
- Backend.
- APIs.
- Database.
- Auth.
- Integrations.
- Deployment planning.

### Frontend and UI Work

Use `frontend-skill`.

Use it for:

- Customer storefront.
- Seller center.
- Admin panel.
- Mobile-responsive layouts.
- Polished marketplace UI.
- Dashboard UX.

### API Design

Use `api-design-ks`.

Use it for:

- REST resource naming.
- Request/response contracts.
- Pagination.
- Filtering.
- Status codes.
- API error format.

### Next.js Work

Use `next-best-practices` if Next.js is selected.

Use it for:

- App Router structure.
- Server/client component boundaries.
- Metadata.
- Route handlers.
- Image/font optimization.
- Data loading.

### React Refactors

Use `react-best-practices` after editing multiple TSX components.

Use it for:

- Hooks review.
- Component structure.
- Accessibility.
- State management.
- Rendering performance.

### Bug Fixing

Use `debugger`.

Use it for:

- Runtime errors.
- Broken flows.
- Build failures.
- API bugs.
- Database errors.

### Production Review

Use `code-reviewer`.

Use it for:

- Security review.
- Performance review.
- Production readiness.
- Missing tests.
- Risk report.

### End-to-End Verification

Use `verification`.

Use it when:

- Dev server is running.
- Browser flow must be checked.
- API and database path must be verified.
- User asks "check all", "complete checklist", or "is it ready".

## 3. Project Local Skill

This workspace includes a local project skill:

`.agents/skills/beeshub-marketplace/SKILL.md`

Use it as the 1HandIndia-specific operating guide. It captures the project goal, scope, build quality target, and read order.

## 4. Before Coding Checklist

- Read project context.
- Confirm the locked Phase 1 technology stack.
- Read `docs/IndiHub_FINAL_TECH_STACK_LOCK.md`.
- Confirm folder structure.
- Confirm database provider.
- Confirm auth strategy.
- Confirm initial implementation milestone.
- Confirm frozen web-first Phase 1 scope.
- Confirm INR 200,000 milestone timeline.
